# DataMerge 训练可视化使用指南

## 快速启动

### Windows
```bash
start_web.bat
```

### Linux/Mac
```bash
./start_web.sh
```

启动后，在浏览器中访问：http://localhost:8000

## 功能说明

### 1. 训练配置
- **租户ID**: 输入您的租户标识符（例如：company_a）
- **源文件**: 上传一个或多个源数据Excel文件
- **目标文件**: 上传预期的结果Excel文件
- **AI模型**: 选择使用的AI模型
  - Claude（推荐）：准确率高，支持Prompt Caching
  - OpenAI GPT-4：通用性强
  - DeepSeek：成本较低
- **生成模式**:
  - 自动选择：系统根据复杂度自动选择
  - 模块化模式：适合复杂数据处理
  - 公式模式：适合简单计算
- **最大迭代次数**: 训练的最大尝试次数（1-10次）

### 2. 实时进度监控
训练开始后，您可以实时查看：
- **当前状态**: 显示训练的当前阶段
- **进度条**: 可视化显示整体进度
- **当前迭代**: 显示当前是第几次迭代
- **已用时间**: 显示训练已经运行的时间
- **准确率**: 显示当前的准确率

### 3. 迭代历史
每次迭代都会显示：
- 迭代编号
- 执行状态（成功/失败/进行中）
- 准确率（如果成功）

### 4. 实时日志
在日志窗口中可以看到：
- 训练的详细步骤
- AI生成代码的过程
- 错误信息和警告
- 系统状态更新

### 5. 训练结果
训练完成后显示：
- 训练状态（成功/失败）
- 脚本ID
- 总迭代次数
- 最终准确率
- 训练总时长

### 6. 操作按钮
- **重新训练**: 清空当前结果，重新开始训练
- **下载脚本**: 下载训练好的Python脚本

## API接口

### 流式训练API
```
POST /api/train/stream
Content-Type: multipart/form-data

参数:
- tenant_id: 租户ID
- source_files: 源文件列表
- target_file: 目标文件
- ai_provider: AI提供商 (claude/openai/deepseek)
- mode: 生成模式 (auto/modular/formula)
- max_iterations: 最大迭代次数

返回: Server-Sent Events (SSE) 流
```

### 下载脚本API
```
GET /api/download-script/{tenant_id}

返回: Python脚本文件
```

### 获取训练状态API
```
GET /api/training-status/{tenant_id}

返回: JSON格式的训练状态信息
```

### 获取训练日志API
```
GET /api/training-logs/{tenant_id}?limit=10

返回: JSON格式的训练日志列表
```

## 技术架构

### 前端
- 纯HTML/CSS/JavaScript
- Server-Sent Events (SSE) 实时通信
- 响应式设计

### 后端
- FastAPI框架
- 异步流式响应
- 事件驱动架构

### 通信流程
```
浏览器 → POST /api/train/stream → 后端开始训练
                                    ↓
                            实时推送训练事件
                                    ↓
浏览器 ← SSE事件流 ← 训练进度/日志/结果
```

## 事件类型

训练过程中会推送以下类型的事件：

1. **status**: 状态更新
2. **iteration_start**: 迭代开始
3. **iteration_progress**: 迭代进度
4. **iteration_complete**: 迭代完成
5. **log**: 日志消息
6. **complete**: 训练完成
7. **error**: 错误信息

## 故障排除

### 问题：页面无法加载
- 检查服务器是否正常启动
- 确认端口8000未被占用
- 查看控制台日志

### 问题：训练无法开始
- 确认所有必填字段已填写
- 检查文件格式是否为.xlsx或.xls
- 查看浏览器控制台的错误信息

### 问题：训练中断
- 检查网络连接
- 查看后端日志
- 确认AI API密钥配置正确

### 问题：准确率低
- 增加最大迭代次数
- 尝试不同的AI模型
- 检查源文件和目标文件的格式是否正确

## 注意事项

1. 训练过程中请勿关闭浏览器标签页
2. 大文件训练可能需要较长时间
3. 确保.env文件中配置了正确的AI API密钥
4. 建议使用Chrome或Firefox浏览器以获得最佳体验

## 示例

### 基本训练流程
1. 打开 http://localhost:8000
2. 输入租户ID：`company_a`
3. 上传源文件：`salary_base.xlsx`, `attendance.xlsx`
4. 上传目标文件：`salary_result.xlsx`
5. 选择AI模型：`Claude`
6. 选择模式：`自动选择`
7. 设置迭代次数：`5`
8. 点击"开始训练"
9. 观察实时进度和日志
10. 训练完成后下载脚本
