# 并发问题分析和解决方案

## 问题分析

### 当前系统的并发情况

系统使用FastAPI + uvicorn，支持异步处理，但存在以下并发风险：

#### 1. **文件写入竞争（高风险）**

**问题场景**：
- 用户A和用户B同时训练同一个租户（如"达美乐70"）
- 两个请求同时写入 `tenants/达美乐70/best_history.json`
- 可能导致文件损坏或数据丢失

**代码位置**：
```python
# ai_engine/training_engine.py 第184-207行
def _save_historical_best(self, tenant_id: str, score: float, code: str) -> None:
    history_path = self._get_best_history_path(tenant_id)
    history_path.parent.mkdir(parents=True, exist_ok=True)

    # ❌ 没有文件锁，多个进程同时写入会冲突
    with open(history_path, 'w', encoding='utf-8') as f:
        json.dump(history_data, f, ensure_ascii=False, indent=2)
```

#### 2. **脚本文件名冲突（中风险）**

**问题场景**：
- 两个训练任务同时进行到第3次迭代
- 生成的文件名可能相同：`script_03_0.95_abc12345.py`
- 后写入的会覆盖先写入的

**代码位置**：
```python
# ai_engine/training_engine.py 第1211-1245行
def _save_generated_script(self, code: str, iteration: int, tenant_id: str, score: float = 0.0) -> str:
    # ❌ 文件名只基于迭代次数和代码哈希，可能冲突
    filename = f"script_{iteration:02d}_{score_str}_{code_hash}.py"
```

#### 3. **日志文件竞争（中风险）**

**问题场景**：
- 多个训练任务同时写入同一个日志文件
- 日志可能交错或丢失

#### 4. **API限流问题（低风险）**

**问题场景**：
- 多个训练任务同时调用Claude API
- 可能触发API限流（Rate Limit）
- 导致所有训练任务失败

### 不会卡死的原因

✅ **好消息**：系统不会"卡死"，因为：

1. **租户隔离**：每个租户有独立的文件夹
   - `tenants/达美乐70/`
   - `tenants/rex104/`
   - 不同租户之间不会冲突

2. **异步处理**：FastAPI使用异步，不会阻塞
   - 多个请求可以并发处理
   - 不会因为一个请求慢而影响其他请求

3. **无全局锁**：没有使用全局锁或单例模式
   - 每个请求独立处理
   - 不会互相等待

### 会出现的问题

❌ **坏消息**：虽然不会卡死，但会出现以下问题：

1. **同一租户并发训练**：
   - 文件覆盖
   - 历史记录混乱
   - 结果不可预测

2. **多租户并发训练**：
   - API限流
   - 响应变慢
   - 可能部分失败

## 解决方案

### 方案1：添加文件锁（推荐）

使用文件锁防止并发写入冲突。

#### 实现方式

```python
import fcntl  # Linux/Mac
# 或
import msvcrt  # Windows

def _save_historical_best_with_lock(self, tenant_id: str, score: float, code: str) -> None:
    """保存历史最佳分数（带文件锁）"""
    history_path = self._get_best_history_path(tenant_id)
    history_path.parent.mkdir(parents=True, exist_ok=True)

    from datetime import datetime
    history_data = {
        "best_score": score,
        "best_code": code,
        "updated_at": datetime.now().isoformat()
    }

    try:
        # 使用文件锁
        with open(history_path, 'w', encoding='utf-8') as f:
            # Linux/Mac
            fcntl.flock(f.fileno(), fcntl.LOCK_EX)
            try:
                json.dump(history_data, f, ensure_ascii=False, indent=2)
            finally:
                fcntl.flock(f.fileno(), fcntl.LOCK_UN)

        self.logger.info(f"历史最佳分数已保存: {score:.2%}")
    except Exception as e:
        self.logger.warning(f"保存历史最佳分数失败: {e}")
```

**优点**：
- ✅ 完全防止文件冲突
- ✅ 跨进程有效
- ✅ 系统级保护

**缺点**：
- ❌ 需要处理跨平台兼容性（Linux/Windows）
- ❌ 增加代码复杂度

### 方案2：添加训练锁（推荐）

为每个租户添加训练锁，同一租户同时只能有一个训练任务。

#### 实现方式

```python
import threading
from collections import defaultdict

# 全局训练锁字典
_training_locks = defaultdict(threading.Lock)

@app.post("/api/train")
async def train_model(
    tenant_id: str = Form(...),
    # ...
):
    # 获取租户的训练锁
    lock = _training_locks[tenant_id]

    # 尝试获取锁（非阻塞）
    if not lock.acquire(blocking=False):
        raise HTTPException(
            status_code=409,
            detail=f"租户 {tenant_id} 正在训练中，请等待当前训练完成"
        )

    try:
        # 执行训练
        result = await training_engine.train(...)
        return result
    finally:
        # 释放锁
        lock.release()
```

**优点**：
- ✅ 简单易实现
- ✅ 防止同一租户并发训练
- ✅ 用户体验好（明确提示）

**缺点**：
- ❌ 只在单进程有效（如果使用多worker会失效）
- ❌ 需要处理锁泄漏

### 方案3：使用唯一文件名（推荐）

为每个训练任务生成唯一的文件名。

#### 实现方式

```python
def _save_generated_script(self, code: str, iteration: int, tenant_id: str, score: float = 0.0) -> str:
    """保存生成的脚本到文件（使用唯一文件名）"""
    from pathlib import Path
    import hashlib
    import time
    import uuid

    scripts_dir = Path("tenants") / tenant_id / "scripts"
    scripts_dir.mkdir(parents=True, exist_ok=True)

    # 生成唯一文件名：时间戳_UUID_哈希值.py
    timestamp = int(time.time() * 1000)  # 毫秒级时间戳
    unique_id = str(uuid.uuid4())[:8]
    code_hash = hashlib.md5(code.encode('utf-8')).hexdigest()[:8]
    score_str = f"{score:.2f}" if score > 0 else "0.00"

    filename = f"script_{timestamp}_{unique_id}_{code_hash}.py"
    file_path = scripts_dir / filename

    try:
        with open(file_path, 'w', encoding='utf-8') as f:
            f.write(code)

        self.training_logger.log_info(f"保存生成的脚本: {filename}")
        return str(file_path)
    except Exception as e:
        self.training_logger.log_error(f"保存脚本失败: {e}")
        return ""
```

**优点**：
- ✅ 完全避免文件名冲突
- ✅ 简单易实现
- ✅ 不影响现有逻辑

**缺点**：
- ❌ 文件名不够直观

### 方案4：API限流保护

限制并发API调用数量。

#### 实现方式

```python
import asyncio
from asyncio import Semaphore

# 全局API调用信号量（最多3个并发）
_api_semaphore = Semaphore(3)

async def generate_code_with_rate_limit(self, prompt: str) -> str:
    """生成代码（带限流保护）"""
    async with _api_semaphore:
        # 调用AI生成代码
        return self.ai_provider.generate_code(prompt)
```

**优点**：
- ✅ 防止API限流
- ✅ 保护系统稳定性

**缺点**：
- ❌ 可能增加等待时间

## 推荐方案组合

### 最佳实践（3个方案组合）

1. **方案2：添加训练锁** - 防止同一租户并发训练
2. **方案3：使用唯一文件名** - 防止文件名冲突
3. **方案4：API限流保护** - 防止API限流

### 实现优先级

#### 高优先级（必须实现）

1. ✅ **添加训练锁** - 防止同一租户并发训练
   - 简单有效
   - 用户体验好
   - 避免大部分问题

#### 中优先级（建议实现）

2. ✅ **使用唯一文件名** - 防止文件名冲突
   - 简单易实现
   - 不影响现有逻辑

3. ✅ **API限流保护** - 防止API限流
   - 保护系统稳定性
   - 避免所有训练失败

#### 低优先级（可选）

4. ⭕ **添加文件锁** - 防止文件写入冲突
   - 如果实现了训练锁，这个就不太需要了
   - 增加复杂度

## 当前系统的并发能力

### 理论并发能力

- **不同租户**：可以同时训练，理论上无限制
- **同一租户**：❌ 不建议并发，会出现文件冲突

### 实际建议

- **小规模（<10个租户）**：当前系统可以正常工作
- **中规模（10-50个租户）**：建议添加训练锁
- **大规模（>50个租户）**：建议添加训练锁 + API限流

## 总结

### 当前状态

- ✅ 不会卡死
- ❌ 同一租户并发训练会有问题
- ⚠️ 多租户并发可能触发API限流

### 建议改进

1. **立即实现**：添加训练锁（防止同一租户并发）
2. **尽快实现**：使用唯一文件名（防止文件冲突）
3. **考虑实现**：API限流保护（防止API限流）

### 风险评估

- **低风险**：如果用户不会同时训练同一个租户
- **中风险**：如果有多个用户可能同时操作
- **高风险**：如果是生产环境，多用户高并发

现在系统可以处理不同租户的并发训练，但同一租户的并发训练需要添加保护机制！
