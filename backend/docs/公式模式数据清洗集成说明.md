## 工作流程

### 数据清洗流程

```
1. 加载源数据
   ↓
   source_data = {"文件名_sheet名": {"df": DataFrame, "columns": [列名]}}

2. 应用数据清洗（AI生成的clean_source_data函数）
   ↓
   - 过滤重复工号
   - 过滤空值
   - 过滤离职员工
   - 过滤不符合条件的记录
   ↓
   cleaned_source_data

3. 写入Excel（write_source_sheets）
   ↓
   source_sheets = {"文件名_sheet名": {"df": DataFrame, "ws": worksheet}}

4. 填充结果sheet（AI生成的fill_result_sheets函数）
   ↓
   使用VLOOKUP引用清洗后的源数据
```

## 使用方法

### 1. 在规则文档中编写清洗规则

```markdown
## 六、数据清洗规则

- 【员工基础表】，工号重复的数据不能作为计薪雇员。
- 【员工基础表】，工号或姓名为空的不能作为计薪雇员。
- 【员工基础表】，员工状态必须为"在职"，否则不能作为计薪雇员。
- 【考勤表】，工号或姓名为空的不能参与考勤统计。
- 【考勤表】，工号对应的姓名必须与员工基础表中工号对应的姓名一致。
```

## 优势

1. **透明可见**：清洗逻辑在生成的代码中清晰可见
2. **灵活可调**：用户可以直接修改清洗逻辑
3. **性能优化**：在写入Excel之前清洗，避免处理无效数据
4. **公式正确**：VLOOKUP引用的是清洗后的数据，确保结果准确

## 注意事项

1. **清洗时机**：数据清洗在写入Excel之前进行，确保Excel中的源数据已经是清洗后的
2. **函数可选**：如果规则文档中没有数据清洗规则，AI不会生成 `clean_source_data` 函数
3. **兼容性**：主函数会检查 `clean_source_data` 是否存在，不存在则跳过清洗步骤
4. **数据结构**：清洗函数的输入输出格式必须一致，都是 `{"sheet名": {"df": DataFrame, "columns": [列名]}}`
