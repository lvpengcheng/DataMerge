# 数据清洗和警告信息功能使用说明

## 功能概述

系统现在支持从规则文档中自动提取数据清洗规则和警告规则，并在AI生成的代码中实现这些规则。

## 功能特性

### 1. 数据清洗规则
在复制基础数据时，自动过滤不符合条件的数据，例如：
- 工号重复的数据
- 工号或姓名为空的数据
- 账单供应商不包含"外服"和"全职"的数据
- 其他表中工号对应的姓名与薪资备份表不一致的数据

### 2. 警告信息规则
在处理数据时，检查数据匹配情况并生成警告，例如：
- 社保表中的身份证号码在薪资备份表中不存在
- 薪资备份表中的工作所在地在产假政策表中不存在
- 薪资备份表中的合同所属地在计提残保金规则表中不存在

### 3. 导入校验规则
在导入数据时，验证数据完整性，例如：
- 工号重复检查
- 必填字段检查
- 数据一致性检查

## 规则文档格式

在规则文档（Markdown格式）中，按以下格式编写规则：

### 数据清洗规则章节

```markdown
## 六、数据清洗规则

- 【薪资备份表】，工号重复的数据不能作为计薪雇员。
- 【薪资备份表】，工号或姓名为空的不能作为计薪雇员。
- 【薪资备份表】，账单供应商中必须包含"外服"和"全职"，否则不能作为计薪雇员。
- 【考勤报表】，工号对应的中文姓名必须与薪资备份表中工号对应的中文姓名一致，否则不能作为计薪雇员。
```

### 警告信息章节

```markdown
## 七、警告信息

社保表中的身份证号码必须在薪资备份表中存在，否则需提示"社保表中身份证号码未匹配到薪资备份表"的记录。

薪资备份表中的工作所在地必须在产假政策表中存在，否则需提示"薪资备份表中工作所在地未匹配到产假政策城市"的记录。
```

### 导入校验规则

```markdown
### 源文件1：薪资备份表

**导入校验规则：**
- 若存在重复"工号"，导入失败
- "工号"或"中文姓名"为空，导入失败
- "账单供应商"必须包含外服和全职，否则导入失败
```

## AI生成的代码结构

AI会生成包含以下功能的代码：

### 1. 数据清洗函数

```python
def clean_employee_data(df, source_name="数据表"):
    """
    清洗员工数据，过滤不符合条件的记录

    Args:
        df: 原始数据DataFrame
        source_name: 数据来源名称

    Returns:
        清洗后的DataFrame
    """
    warnings = []
    original_count = len(df)

    # 检查工号重复
    duplicates = df[df.duplicated(subset=['工号'], keep=False)]
    if len(duplicates) > 0:
        warnings.append(f"{source_name}: 发现{len(duplicates)}条工号重复的记录，已过滤")
        df = df.drop_duplicates(subset=['工号'], keep='first')

    # 检查工号或姓名为空
    empty_records = df[(df['工号'].isna()) | (df['姓名'].isna())]
    if len(empty_records) > 0:
        warnings.append(f"{source_name}: 发现{len(empty_records)}条工号或姓名为空的记录，已过滤")
        df = df.dropna(subset=['工号', '姓名'])

    # 检查账单供应商
    if '账单供应商' in df.columns:
        invalid_supplier = df[~df['账单供应商'].str.contains('外服|全职', na=False)]
        if len(invalid_supplier) > 0:
            warnings.append(f"{source_name}: 发现{len(invalid_supplier)}条账单供应商不符合要求的记录，已过滤")
            df = df[df['账单供应商'].str.contains('外服|全职', na=False)]

    cleaned_count = len(df)
    if original_count != cleaned_count:
        warnings.append(f"{source_name}: 原始记录{original_count}条，清洗后{cleaned_count}条")

    return df, warnings
```

### 2. 警告检查函数

```python
def check_data_warnings(base_df, social_df, policy_df):
    """
    检查数据匹配情况，生成警告信息

    Args:
        base_df: 薪资备份表
        social_df: 社保表
        policy_df: 产假政策表

    Returns:
        警告信息列表
    """
    warnings = []

    # 检查社保表中的身份证号码是否在薪资备份表中存在
    if social_df is not None and len(social_df) > 0:
        unmatched_ids = social_df[~social_df['身份证号码'].isin(base_df['身份证'])]
        if len(unmatched_ids) > 0:
            warnings.append(f"警告: 社保表中有{len(unmatched_ids)}条身份证号码未匹配到薪资备份表")
            for idx, row in unmatched_ids.iterrows():
                warnings.append(f"  - 身份证号码: {row['身份证号码']}")

    # 检查工作所在地是否在产假政策表中存在
    if policy_df is not None and len(policy_df) > 0:
        unmatched_cities = base_df[~base_df['工作所在地'].isin(policy_df['城市'])]
        if len(unmatched_cities) > 0:
            unique_cities = unmatched_cities['工作所在地'].unique()
            warnings.append(f"警告: 薪资备份表中有{len(unique_cities)}个工作所在地未匹配到产假政策城市")
            for city in unique_cities:
                count = len(unmatched_cities[unmatched_cities['工作所在地'] == city])
                warnings.append(f"  - {city}: {count}人")

    return warnings
```

## 使用流程

### 1. 编写规则文档
在规则文档中按照上述格式编写数据清洗规则和警告规则。

### 2. 训练脚本
系统会自动从规则文档中提取这些规则，并在生成代码时包含相应的数据清洗和警告检查逻辑。

### 3. 执行计算
执行生成的脚本时，会自动应用数据清洗规则，并在计算完成后返回警告信息。

### 4. 查看结果
在计算结果中可以看到：
- 清洗了多少条不符合条件的数据
- 哪些数据存在匹配问题
- 具体的警告信息列表

## 注意事项

1. **规则格式**: 规则文档必须按照指定格式编写，系统才能正确提取
2. **表名标识**: 使用【表名】格式标识涉及的表
3. **字段名**: 使用引号或直接文本标识字段名
4. **警告信息**: 警告不会阻止计算，只是提示用户注意
5. **数据清洗**: 清洗规则会直接过滤数据，被过滤的数据不参与计算

## 技术实现

### 核心组件

1. **RuleExtractor**: 规则提取器，从Markdown文档中提取结构化规则
2. **PromptGenerator**: 提示词生成器，将规则整合到AI提示词中
3. **AI代码生成**: AI根据规则生成包含清洗和警告逻辑的代码

### 工作流程

```
规则文档 → RuleExtractor → 结构化规则 → PromptGenerator → AI提示词 → AI生成代码 → 执行并返回警告
```
