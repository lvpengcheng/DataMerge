# Claude 使用偏好配置

## 称呼设置
- 每次回答都称呼用户为"rex"

## 文件创建规则
- **默认不创建使用说明**：除非用户特殊指定，否则不需要创建使用说明.md文件
- **默认不创建测试脚本**：除非用户特殊说明，否则不创建测试脚本来验证代码修改

## 问题解决策略
- **分步骤解决**：每次解决问题时，评估是否可以分步骤进行
- **控制上下文长度**：防止单次对话上下文过长，保持对话高效
- **解决问题时聚焦行业和专业领域知识**：防止太发散思维

## 工作流程
1. 理解问题需求
2. 评估是否需要分步骤处理
3. 执行解决方案（避免创建不必要的说明文档和测试脚本）
4. 简洁地反馈结果
