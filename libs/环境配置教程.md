## 2. 安装 Python

### 2.1 下载 Python

1. 访问 Python 官网：https://www.python.org/downloads/
2. 下载 **Python 3.12.0+** 的 Windows installer (64-bit)

### 2.2 安装 Python

1. 运行安装程序
2. **务必勾选** `Add Python to PATH`
3. 点击 `Install Now`


### 2.3 验证安装

打开命令提示符（CMD）或 PowerShell，运行：

```powershell
python --version
```

应显示类似：`Python 3.12.0`

确认是 64 位版本：

```powershell
python -c "import struct; print(struct.calcsize('P') * 8)"
```

应显示：`64`

## 3. 安装 .NET 运行时

### 3.1 下载 .NET 8.0 SDK/运行时

1. 访问 .NET 下载页面：https://dotnet.microsoft.com/download/dotnet/8.0
2. 下载 **.NET Desktop Runtime 8.0.x (Windows x64)**
   - 或者安装完整的 **SDK**（包含运行时，推荐开发者使用）

> [!NOTE]
> 本项目使用 `Microsoft.WindowsDesktop.App` 框架，因此必须安装 **Desktop Runtime**，不是仅安装 ASP.NET Core Runtime。

### 3.2 验证安装

打开命令提示符，运行：

```powershell
dotnet --list-runtimes
```

应看到类似输出：

```
Microsoft.AspNetCore.App 8.0.23 [C:\Program Files\dotnet\shared\Microsoft.AspNetCore.App]
Microsoft.NETCore.App 8.0.23 [C:\Program Files\dotnet\shared\Microsoft.NETCore.App]
Microsoft.WindowsDesktop.App 8.0.23 [C:\Program Files\dotnet\shared\Microsoft.WindowsDesktop.App]
```

这里 `8.0.23` 就是 **Microsoft.WindowsDesktop.App** 的版本号


## 6. 获取 DLL 文件

### 6.1 Aspose.Words.dll



**直接在从网盘下载的对应版本的压缩包或者文件夹中的 `lib/net8.0/` 目录下找到 `Aspose.Words.dll`**



### 6.2 SkiaSharp.dll 和 libSkiaSharp.dll

1. 访问 https://www.nuget.org/packages/SkiaSharp/
2. 下载 `.nupkg` 文件并解压
3. 托管程序集：`lib/net8.0/SkiaSharp.dll`
4. 本地库：`runtimes/win-x64/native/libSkiaSharp.dll`

> [!CAUTION]
> **libSkiaSharp.dll** 是本地 DLL，必须使用 `win-x64` 版本，路径在 `runtimes/win-x64/native/` 中！

### 6.3 System.Text.Encoding.CodePages.dll

1. 访问 https://www.nuget.org/packages/System.Text.Encoding.CodePages/
2. 下载并解压，在 `lib/net8.0/` 目录下找到对应 DLL

### 6.4 Aspose.Total.NET.lic

**直接在从网盘下载的对应版本的压缩包或者文件夹中找到 `Aspose.Total.NET.lic`**

## 7. 配置 runtimeconfig.json

在 `libs` 目录下创建 `runtimeconfig.json` 文件：

```json
{
  "runtimeOptions": {
    "tfm": "net8.0",
    "framework": {
      "name": "Microsoft.WindowsDesktop.App",
      "version": "8.0.23"
    }
  }
}
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `tfm` | Target Framework Moniker，指定 .NET 版本 |
| `framework.name` | 使用的框架类型 |
| `framework.version` | 对应 .NET Runtime 的版本号 |

> [!IMPORTANT]
> `version` 的值必须对应你安装的 **.NET Runtime 版本**，例如安装的是 8.0.23 就填 `"8.0.23"`。

**查看已安装的 Runtime 版本：**

```powershell
# Windows
dotnet --list-runtimes | findstr "WindowsDesktop"

# macOS / Linux
dotnet --list-runtimes | grep "WindowsDesktop"
```

输出示例：
```
Microsoft.WindowsDesktop.App 8.0.23 [C:\Program Files\dotnet\shared\Microsoft.WindowsDesktop.App]
```

将 `version` 值替换为你实际安装的版本号。

## 9. 常见问题与解决方案

### 问题 1：`RuntimeError: Failed to create a default .NET runtime`

**原因：** .NET 运行时未安装或版本不匹配

**解决方案：**
1. 确认已安装 .NET Desktop Runtime 8.0
2. 检查 `runtimeconfig.json` 中的版本号是否与已安装版本一致
3. 运行 `dotnet --list-runtimes` 确认

---

### 问题 2：`System.IO.FileNotFoundException: SkiaSharp`

**原因：** SkiaSharp.dll 未正确加载

**解决方案：**
1. 确认 `SkiaSharp.dll` 在 `libs` 目录中
2. 确认代码中先加载 SkiaSharp 再加载 Aspose.Words

---

### 问题 3：PDF 中图片显示为空白

**原因：** `libSkiaSharp.dll`（本地库）未被加载

**解决方案：**

这是最常见的问题！必须在导入 clr 之前调用：

```python
os.add_dll_directory(libs_dir)
```

> [!IMPORTANT]
> Python 3.8+ 在 Windows 上修改了 DLL 搜索策略，不再自动搜索 `PATH` 和当前目录。必须显式调用 `os.add_dll_directory()` 注册目录。

---

### 问题 4：中文乱码

**原因：** .NET Core 默认不包含旧版代码页

**解决方案：**

确保代码中包含：

```python
clr.AddReference("System.Text.Encoding.CodePages")
import System.Text
System.Text.Encoding.RegisterProvider(System.Text.CodePagesEncodingProvider.Instance)
```

并且 `System.Text.Encoding.CodePages.dll` 在 `libs` 目录中。

---

### 问题 5：`Aspose.Words.dll` 找不到

**原因：** DLL 路径错误或未使用绝对路径

**解决方案：**

使用绝对路径加载：

```python
clr.AddReference(os.path.join(libs_dir, "Aspose.Words.dll"))
```

---

### 问题 6：32 位 vs 64 位不匹配

**原因：** Python 是 32 位，但 DLL 是 64 位

**解决方案：**

1. 确认 Python 是 64 位版本
2. 确认下载的是 `win-x64` 版本的 DLL

检查 Python 架构：

```powershell
python -c "import struct; print(struct.calcsize('P') * 8)"
```
